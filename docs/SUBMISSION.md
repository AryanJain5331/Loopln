## Project name

LoopIn — AI-Powered Senior Friend for Students

## Tagline

## Project description

LoopIn addresses the information gap faced by students who do not have strong
senior networks, mentors, or professional circles. Many students miss
internship openings, hackathon deadlines, new developer tools, and important
learning updates not because information is unavailable, but because it is not
filtered for their profile at the right time.

LoopIn acts like an AI senior friend. A student enters their year, tech stack,
goals, interests, and location. The agent then follows a five-step reasoning
pipeline: profile intake, contextual search, relevance reasoning,
personalized synthesis, and action delivery. The final output is a set of
5-7 source-linked cards written in conversational language, each explaining
what the signal is, why it matters to that student, and what to do next.

The project demonstrates how agentic AI can reduce information inequality for
students, especially first-generation students and learners from Tier 2/Tier 3
cities who may not have access to well-connected peer networks.

## Features

- Student profile onboarding
- Visible multi-step reasoning pipeline
- Personalized opportunity ranking
- Hackathon, tool, learning, open-source, and community cards
- Senior-friend tone instead of generic headlines
- Deep-dive view with source proof and recommended next action
- Source links and deadline verification reminders
- Demo mode with curated data
- Optional Microsoft Foundry + Foundry IQ backend path

## Microsoft IQ integration

The live backend is designed to use Foundry IQ as a grounded knowledge source
for LoopIn's ranking and safety rubric. The Search Agent can combine that
rubric with current source retrieval to rank opportunities by student profile,
urgency, actionability, and source quality.

## Reliability and safety

LoopIn keeps source links attached to every recommendation, asks students to
verify deadline-sensitive pages before acting, avoids accounts or sensitive
personal data in demo mode, and keeps cloud credentials server-side.
